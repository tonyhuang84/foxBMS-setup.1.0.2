237 /* ***************************************
238  * CAN Fault Confinement
239  ****************************************/
240 
241 /* Fault confinement is a checking mechanism that makes it possible to distinguish between short disturbances
242  (e.g. switching noise from a nearby power cable couples into the transmission media) and permanent failures
243  (e.g. a node is malfunctioning and disturbs the bus).
244 
245  Manipulation of the error counters is asymmetric. On a successful transmission, or reception, of a message,
246  the respective error counter is decremented if it had not been at zero. In the case of a transmit or receive
247  error the counters are incremented, but by a value greater than the value they would be decrement by following
248  a successful message transaction.
249 
250  If a node detects a local error condition (e.g. due to local conducted noise, application software, etc.),
251  its resulting error flag (primary error flag) will subsequently cause all other nodes to respond with an error
252  flag too (secondary error flags). It is important that a distinction is made between the nodes that detected an
253  error first and the nodes which responded to the primary error flag. If a node transmits an active error frame,
254  and it monitors a dominant bit after the sixth bit of its error flag, it considers itself as the node that has
255  detected the error first. In the case where a node detects errors first too often, it is regarded as malfunctioning,
256  and its impact to the network has to be limited. Therefore, a node can be in one of three possible error states:
257 
258  ERROR ACTIVE - Both of its error counters are less than 128. It takes part fully in bus communication and signals
259  an error by transmission of an active error frame.This consists of sequence of 6 dominant bits followed
260  by 8 recessive bits, all other nodes respond with the appropriate error flag, in response to the
261  violation of the bit stuffing rule.
262 
263  ERROR PASSIVE - A node goes into error passive state if at least one of its error counters is greater than 127. It
264  still takes part in bus activities, but it sends a passive error frame only, on errors. Furthermore,
265  an error passive node has to wait an additional time (Suspend Transmission Field, 8 recessive bits after
266  Intermission Field) after transmission of a message, before it can initiate a new data transfer. The
267  primary passive error flag consists of 6 passive bits and thus is transparent on the bus and will
268  not jam communications.
269 
270  BUS OFF - If the Transmit Error Counter of a CAN controller exceeds 255, it goes into the bus off state. It is
271  disconnected from the bus (using internal logic) and does not take part in bus activities anymore. In order
272  to reconnect the protocol controller, a so-called Bus Off recovery sequence has to be executed. This usually
273  involves the re-initialization and configuration of the CAN controller by the host system, after which it
274  will wait for 128 * 11 recessive bit times before it commences communication.
275 
276  */
277 
278 /* ***************************************
279  * CAN Error Confinement Rules
280  ****************************************/
281 
282 /* REC: Receive Error Counter, TEC: Transmit Error Counter */
283 
284 /* - When a receiver detects an error, the REC will be increased by 1, except when the detected error was a Bit Error
285  during the sending of an Active error Flag or an Overload Flag.
286 
287  - When a receiver detects a dominant bit as the first bit after sending an Error Flag, the REC will be increased by 8.
288 
289  - When a transmitter sends an Error Flag, the TEC is increased by 8.
290  Exception 1: If the transmitter is Error Passive and detects an ACK Error because of not detecting a dominant ACK
291  and does not detect a dominant bit while sending its Passive Error Flag.
292  Exception 2: If the transmitter sends an Error Flag because a Stuff Error occurred during arbitration, and should
293  have been recessive, and has been sent as recessive but monitored as dominant.
294 
295  - If the transmitter detects a Bit Error while sending an Active Error Flag or an Overload Frame, the TEC is increased by 8.
296 
297  - If a receiver detects a Bit Error while sending an Active Error Flag or an Overload Flag, the REC is increased by 8.
298 
299  - Any node tolerates up to 7 consecutive dominant bits after sending an Active Error Flag, Passive Error Flag or Overload Flag.
300  After detecting the fourteenth consecutive dominant bit (in case of an Active Error Flag or an Overload Flag) or after
301  detecting the eighth consecutive dominant bit following a Passive Error Flag, and after each sequence of additional eight
302  consecutive dominant bits, ever y transmitter increases its TEC by 8 and every receiver increases its REC by 8.
303 
304  - After successful transmission of a frame (getting ACK and no error until EOF is finished), the TEC is decreased by 1 unless it was already 0.
305 
306  - After the successful reception of a frame (reception without error up to the ACK Slot and the successful sending of the ACK bit),
307  the REC is decreased by 1, if it was between 1 and 127. If the REC was 0, it stays 0, and if it was greater than 127, then it
308  will be set to a value between 119 and 127.
309 
310  - A node is Error Passive when the TEC equals or exceeds 128, or when the REC equals or exceeds 128. An error condition
311  letting a node become Error Passive causes the node to send an Active Error Flag.
312 
313  - A node is Bus Off when the TEC is greater than or equal to 256.
314 
315  - An Error Passive node becomes Error Active again when both the TEC and the REC are less than or equal to 127.
316 
317  - A node which is Bus Off is permitted to become Error Active (no longer Bus Off) with its error counters both set to 0
318  after 128 occurrence of 11 consecutive recessive bits have been monitored on the bus.
319  */
