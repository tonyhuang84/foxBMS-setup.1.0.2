Module-prefix
EEPR

driver for external EEPROM device AT25128 (16kByte) /HOLD and /WP are permanently kept at Hi-Level, thus HW-Write-Protection (WP) is disabled. SW-Protection(Block Write Protection) could be done by Sending Command EEPR_CMD_WREN if disabled before by EEPR_CMD_WRDI. At startup the device is unprotected. Actual state will be provided by EEPR_CMD_RDSR (Read Status Register). A Block Write Protection can be controlled by Status Register Bit BP1 and BP0 in different address ranges within the whole range: Byte Address 0x0000..0x3FFF

Protected Address BP1 BP0
None 0 0
0x3000..0x3FFFF 0 1
0x2000..0x3FFFF 1 0
0x0000..0x3FFFF 1 1
