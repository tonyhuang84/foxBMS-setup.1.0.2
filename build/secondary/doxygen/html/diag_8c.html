DIAG_EntryWrite adds an error entry.

This function adds an entry to the error buffer. It provides some functionality to prevent duplicates from being logged. Multiple occurring error doesn't get logged anymore after they reached a pre-defined error count.

Parameters
eventIDID of entry
eventOK, NOK or RESET
item_nritem number of event
Returns
0xFF if event is logged, otherwise 0

◆ DIAG_GeneralHandler()

static DIAG_RETURNTYPE_e DIAG_GeneralHandler ( DIAG_CH_ID_e  diag_ch_id,
DIAG_EVENT_e  event,
uint8_t  item_nr 
)
static

DIAG_GeneralHandler provides generic error handling, based on configuration.

This function does all the handling based on the user defined configuration. According to its return value further treatment is left to the calling module itself.

Parameters
diag_ch_idevent ID of the event that has occurred
eventevent that occurred (OK, NOK, RESET)
item_nritem nr of event, to distinguish between different calling locations of the event
Returns
DIAG_STATE_UNINITIALIZED if diag module still uninitialized,
DIAG_HANDLER_RETURN_WRONG_ID if invalid diag id,
DIAG_HANDLER_INVALID_TYPE if diag id doesn't correspond to diag type,
DIAG_HANDLER_RETURN_OK if error/warning occurred but no threshold reached or event = DIAG_EVENT_OK/DIAG_EVENT_RESET,
DIAG_HANDLER_RETURN_ERR_OCCURRED if error threshold reached,
DIAG_HANDLER_RETURN_WARNING_OCCURRED if warning threshold reached,
